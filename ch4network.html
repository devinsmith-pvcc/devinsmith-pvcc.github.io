    Operating Systems Terminology

TERM DEFINITION
Network Classifications
  1. LAN
  2. MAN
  3. WAN
 Local Area Network
  1. Metropolitan Area Network
  2. Wide Area Network
Network Models
  1. Bus Network
  2. Star Network
  1. All connected devices are connected to one main cable or link, called the bus.
  2. A data or communication network in which all nodes are independently connected to one central unit.
Protocols
  1. CSMA/CD Ethernet
  2. CSMA/CA Wifi
  1. Carrier Sense, Multiple Acces with Collision Detection. This Protocol dictates that each message be broadcast to all machines on the bus.
  2. Carrier Sense, Multiple Access with Collision Avoidance. A protocol for carrier transmission in 802.11 networks. It was developed to minimize the potential of a collision occurring when two or more stations send their signals over a data link layer.
Combining Networks
  1. Repeater
  2. Bridge
  3. Switch
  4. Router
  5. Gateway
  1. A Device that passes signals back and forth between the two original buses without considering the meaning of the signals.
  2. Like a repeater, but only passes a message when it sees the intended receiving device.
  3. Like a bridge, but can connect to several buses at once.
  4. Connects a network to the internet.
  5. The point where a network connects to the internet.
Client/Server Models
  1. P2P: peer-to-peer
  2. Distributed System
  3. Cloud Computing
  1. When two computers are connected, and can share resources without going through a server.
  2. A number of independent computers linked by a network.
  3. A huge pool of shared computers on a network can be allocated for use by clients as needed.
Internet Addressing
  1. Tier 1
  2. Tier 2
  3. Tier 3
  1. High-speed, high-capacity, international WANs.
  2. Connected to tier 1, more regional and less potent.
  3. Primarily engaged in delivering Internet access to end customers.
IP Address
  1. ICANN
  2. DNS
  3. TLD: Top-Level Domain
  4. Country-Code TLDs
  1. Internet Corporation for Assigned Names and Numbers. A non-profit corporation established to coordinate the Internet's operation.
  2. Domain Name System. An Internet wide directory system.
  3. Suffixes at the end of websites. Examples: .com, .net, .edu, etc.
  4. Suffixes for country specific websites. Examples: .ca, .au, etc.
Internet Protocols
  1. FTP
  2. HTTP
  3. SMTP
  1. File Transfer Protocol. An application for accessing another computer from a great distance using SSH (Secure Shell Protocol)
  2. Hyper Text Transfer Protocol.
  3. Simple Mail Transfer Protocol. Defines a way that two computers on the network may interact when transmitting an email message from one host to the other.
World Wide Web
  1. URL
  2. Client-side activities
  3. Server-side activities
  1. Uniform Resource Locator. Unique addresses giving to documents on the web.
  2. Refers to everything in a web application that is displayed or takes place on the client (end user device).
  3. Everything that happens on the server, instead of on the client.
Internet Four-Layer Software
  1. Application Layer
  2. Transport Layer
  3. Network Layer
  4. Link Layer
  1. Consusts of software units such as clients and servers that use Internet cimmunication to carry out their tasks.
  2. Application layer uses this layer to send and receive messages over the internet the same way we use a delievery service.
  3. Deccides which direction a packet should be sent at each step along the path through the Internet.
  4. Deals with the communication details particular to the individual network in which the computer resides.
Internet is:
  1. Packet-Switching Network
  2. Fault Tolerant
  3. Redundant
  1. The transfer of small pieces of data across various networks.
  2. The ability of a system to continue operating without interruption.
  3. Design in which a component is duplicated so if it fails there will be a backup.
Cybersecurity
  1. Malware
  2. Virus
  3. Worm
  4. Trojan Horse
  5. Spyware
  6. Phishing
  7. DDos Attack
  1. Malicious software.
  2. Software that infects a computer by inserting itself into programs that already reside in the machine.
  3. An autonomous program that transfers itself through a network, taking up residence in computers and forwarding copies of itself to other computers.
  4. A program the enters the computer system disguised as a desirable program, such as a game or useful utility package, that is wollingly imported by the victim.
  5. Software that collects information about activities at the computer on which it resides and reports that information back to the instigator of the attack.
  6. A technique of obtaining information explicitly by simply asking for it.
  7. The process of overloading a computer with messages.
Protection
  1. Firewall
  2. Spam Filter
  3. Proxy Server
  4. Antivirus Software
  5. Cryptography
  6. Public Key Encryption
  1. A primary prevention techique to filter traffic passing through a point in the network.
  2. Firewall designed to block unwanted emails.
  3. A software unit that acts as an intermediary between a client and a server with the goal of shielding the client from adverse actions of the server.
  4. Software used to detect and remove the presence of known viruses and other malware infections.
  5. The study of sending and receiving messages securely in the presence of adversaries.
  6. Involves techniques by which the encryption and decryption algorithms are known, and one of the keys is known, but the encoded message remains secure.
